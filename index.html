How it knows

Read, not typed

The unfair advantage over standalone tool-selector sites: the add-in sits inside the job. Every auto-filled value is labeled with the from model device and remains overridable.

CAD → material

Workpiece material

The model's physical material maps to an ISO group. AL 6061-T6 pre-selects N — Non-ferrous; the user can pick any combination, and presets follow the most conservative group selected.

CAM → operation

Active setup

The operation type comes from the active CAM setup — a 2D Pocket pre-selects end milling. No process dropdown to hunt through.

Geometry → dimensions

Selected feature

"Read Pocket_3" fills diameter, cut depth, and corner radius from the selected geometry. Machine RPM limits are respected in every preset.

Guided tour

Five moves, in order

The demo script the prototype was built around. Run it top to bottom in the panel above.

  1. Match — the hook

    Paste the pre-filled competitor part number and hit Match it: a 96% Accupro equivalent, spec deltas marked exact or close, $58 cheaper, next-day. Add it to Library and cart.

  2. Find — the wizard

    Watch the auto-fill: ISO N from the model, end milling from the setup. Multi-select P+N, pick brands, hit Read Pocket_3, then flip Cards ↔ Compare and expand "See all 127."

  3. Library — the money slide

    Run Extract & Match on the document's tooling: 4 of 5 matched, flip an alternate, watch the savings roll-up update live, stage four tools.

  4. Crib — the shop

    Switch profiles, note the below-minimum reds, then Generate PO — MSC SKUs only, with non-MSC lines explicitly excluded.

  5. Cart — make it real

    Review quantities, honor the crib check ("use crib stock first"), then commit: order, quote, or send everything to the Library instead.

Prototype scope

What's simulated

Data

No live catalog, pricing, or inventory APIs. Every SKU, part number, price, lead time, and account number on this page is fictional, in realistic formats.

Fusion integration

Material read, feature selection, and tool-library write are simulated behaviors. No actual Fusion 360 API calls are made.

Parked

Authentication, multi-user crib sync, punchout/e-procurement, and the tool assembly builder (cutter + collet + holder) — designed, and retained as a future tab.
